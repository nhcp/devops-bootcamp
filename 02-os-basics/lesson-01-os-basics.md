# Module 2, Lesson 01: Introduction to Operating Systems

## 🎯 Objective
Understand the role of the Operating System (OS) as the foundation for infrastructure management.

## 🏗️ What is an Operating System?
An **Operating System** is the primary software that manages a computer's hardware (CPU, RAM, Storage) and provides the environment for applications to run.

## 🧠 Core OS Architecture
* **The Kernel**: The core "brain" of the OS. It sits between the hardware and software, managing system resources and hardware communication.
* **User Space**: The isolated area where user applications (like web browsers, terminals, and Nginx) run. This isolation ensures that an application crash doesn't take down the entire system.
* **System Calls**: The method used by applications in the User Space to request resources from the Kernel.

## 🚀 DevOps Connection
DevOps engineers are responsible for preparing and configuring the servers (infrastructure) where applications live. Mastering the **Command Line Interface (CLI)** is a non-negotiable prerequisite for this role.
