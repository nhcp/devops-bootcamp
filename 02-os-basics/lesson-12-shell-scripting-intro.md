# Module 2, Lesson 12: Shell Scripting Intro

## 🧠 Part 1: Important Basic Theories
Shell scripting is the first step toward DevOps automation[cite: 247, 257].

### 1. What is a script?
A file containing a list of commands that the shell runs automatically[cite: 247].

### 2. The Shebang
The first line `#!/bin/bash` tells Linux to use the Bash shell to run the script.

## 🎙️ Part 2: 15 Essential Interview Questions & Answers

| # | Question | Basic Technical Answer |
| :--- | :--- | :--- |
| **1** | **What is a Shell Script?** | A file that automates Linux commands. |
| **2** | **What is the Shebang?** | `#!/bin/bash`. |
| **3** | **How to run a script?** | `./script.sh` or `bash script.sh`. |
| **4** | **How to make it executable?** | `chmod +x script.sh`. |
| **5** | **What is a variable?** | `NAME="Value"`. |
| **6** | **How to use a variable?** | `echo $NAME`. |
| **7** | **How to take user input?** | `read`. |
| **8** | **What is $1?** | The first argument passed to the script. |
| **9** | **What is $0?** | The name of the script. |
| **10** | **What is $?** | The exit status (0 = success). |
| **11** | **How to add comments?** | Use `#`. |
| **12** | **What is a loop?** | Repeating an action multiple times. |
| **13** | **What is an if-statement?** | Running code based on a condition. |
| **14** | **Why automate?** | To save time and prevent human error. |
| **15** | **File extension?** | `.sh`. |
